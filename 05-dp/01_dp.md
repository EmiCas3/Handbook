# Programación Dinamica
#### Identificar el subproblema
- Encuentra la **pequeña versión del problema** que puedes resolver y que se reutiliza.
- Pregúntate: "Si conozco la solución a los casos más pequeños, ¿puedo construir la solución al caso grande?"
- Ejemplo: En Coin Change, subproblema = "mínimo número de monedas para sumar x".


#### Formular la recurrencia
- Escribe cómo se relaciona un caso con los casos más pequeños.
- Generalmente tiene la forma:
$$
\text{dp}[x] = \text{función de dp[casos más pequeños]}
$$
- Ejemplo: 
$$
\text{solve}(x) = \min_{c \in coins} (\text{solve}(x-c)+1)
$$


#### Definir casos base
- Piensa en **valores triviales** que no dependen de otros subproblemas.
- Ejemplo Coin Change:
  - `solve(0) = 0` → no necesitas monedas para sumar 0
  - `solve(x<0) = ∞` → imposible

#### Memoización o tabla iterativa
- **Memoización:** Guardar resultados de subproblemas recursivos en un array.
- **Iterativa (bottom-up):** Llenar un array desde los casos base hasta el objetivo.
- Ventaja iterativa: menos sobrecarga de llamadas recursivas y más control sobre complejidad.

#### Reconstrucción de la solución
- A veces necesitamos **no solo el valor óptimo**, sino **cómo construirlo**.
- Guardar información adicional en arrays auxiliares, por ejemplo:
  - `first[x]` → primer paso para llegar a x
- Luego se recorre hacia atrás para reconstruir la solución.

#### Contar soluciones
- A veces queremos **todas las formas posibles** de alcanzar un estado.
- La recurrencia cambia de "min" o "max" a "suma":
$$
\text{count}[x] = \sum_{c \in coins} \text{count}[x-c]
$$

#### Optimización de espacio
- Muchos DP pueden reducir el tamaño del array si solo dependen de los últimos k valores.
- Ejemplo: Coin Change solo depende de `dp[x-c]` → se puede usar un vector de tamaño n+1.

### Patrones Tipicos

### Lineal (1D DP)
Subproblema depende de un solo parámetro, ej. suma o longitud.
Ejemplo: Coin Change (mínimo número de monedas para sumar x)

Recursivo:
$$
\text{solve}(x) =
\begin{cases}
\infty & x < 0 \\\\
0 & x = 0 \\\\
\min_{c \in coins} (\text{solve}(x-c)+1) & x > 0
\end{cases}
$$

```cpp
#define INF 1e9
vi value(n+1, INF);
value[0] = 0;
forn (x, 1, n+1) {
    for (auto c : coins) if (x-c >= 0)
        value[x] = min(value[x], value[x-c]+1);
}
```

## Conjuntos (Bitmask DP)
**Ejemplo:** Traveling Salesman Problem (TSP) – recorrer todos los nodos con mínimo costo.
**Idea:** Representamos cada subconjunto con un bitmask; `dp[mask][i]` = mínimo costo para llegar a nodo `i` visitando los nodos de `mask`.

Recursivo:
$$
\text{dp}[mask][i] = \min_{j \in mask, j\ne i} (\text{dp}[mask\setminus\{i\}][j] + cost[j][i])
$$

```cpp
int dp[1<<n][n];
forn (int mask=0; mask<(1<<n); mask++)
    forn (i, 0, n)
        dp[mask][i] = INF;

dp[1][0] = 0; // start at node 0
for(int mask=1; mask<(1<<n); mask++)
{
    forn(u,0,n)
    {
        if(mask & (1<<u))
        {
            forn(v,0,n)
            {
                if(mask & (1<<v) && u!=v)
                    dp[mask][u] = min(dp[mask][u], dp[mask^(1<<u)][v]+cost[v][u]);
            }
        }
    }
}
```

## Intervalos
Subproblema depende de un rango `[i,j]`.
**Ejemplo:** Matrix Chain Multiplication – minimizar costo de multiplicación de matrices.
**Idea:** Para cada intervalo `[i,j]`, probamos todos los puntos `k` donde podemos dividir el problema en dos subintervalos.

Recursivo:
$$
\text{dp}[i][j] =
\begin{cases}
0 & i = j \\\\
\min_{k=i}^{j-1} (\text{dp}[i][k] + \text{dp}[k+1][j] + cost[i][k][j]) & i < j
\end{cases}
$$

```cpp
vvi dp(n, vi(n, 0));
forn (len,2, n+1)
{
    for(int i=0; i<=n-len; i++)
    {
        int j=i+len-1;
        dp[i][j] = INF;
        forn(k, i, j)
            dp[i][j] = min(dp[i][j], dp[i][k]+dp[k+1][j]+cost(i,k,j));
    }
}
```

## Longest Increasing Subsequence (LIS)
- Encontrar la subsecuencia creciente más larga en un arreglo.
- Subproblema: `length(k)` = largo de la LIS que termina en `k`.

Recursivo:
$$
length(k) = 
\begin{cases}
1 & \text{si no existe } i<k \text{ con } array[i]<array[k] \\\\
\max_{i<k, array[i]<array[k]} (length(i)+1) & \text{en otro caso}
\end{cases}
$$

```cpp
vi len(n,1);
forn (k,0,n)
{
    forn(i,0,k)
    {
        if(v[i]<v[k])
            len[k] = max(len[k], len[i]+1);
    }
}
int LIS = *max_element(all(len));
```

## Paths in a Grid
- Encontrar la ruta de suma máxima desde la esquina superior izquierda a la inferior derecha, solo movimientos hacia abajo o derecha.
- Subproblema: `sum(y,x)` = suma máxima hasta la celda `(y,x)`.

Recursivo:
$$
sum(y,x) = max(sum(y-1,x), sum(y,x-1)) + value[y][x]
$$

```cpp
forn(y,1,n+1)
{
    for(x, 1, n+1)
        sum[y][x] = max(sum[y-1][x], sum[y][x-1]) + value[y][x];
}
```

## Knapsack
- Problema: Dado un conjunto de objetos con pesos $w[i]$ y valores $v[i]$, encontrar el subconjunto de máximo valor sin superar capacidad $c$.
- Subproblema: $dp[i][x]$ = máximo valor usando los primeros $i$ objetos con capacidad $x$.

Recursivo:
$$
dp[i][x] =
\begin{cases}
dp[i-1][x] & \text{si no usamos el objeto i} \\\\
dp[i-1][x-w[i]] + v[i] & \text{si usamos el objeto i (x \ge w[i])}
\end{cases}
$$

```cpp
struct Item
{
    int val, w;
};


pair<vvi, vvb> knapsack(vector<Item>&v, int k)
{
    int n = v.size();
    vvi dp(n+1, vi(k+1, 0));
    vvb take(n+1, vb(k+1, false));

    forn(i, 1, n+1)
    {
        forn(c, 0, k+1)
        {
            dp[i][c] = dp[i-1][c]; // No tomar
            if (c >= v[i-1].w && 
                dp[i-1][c - v[i-1].w] + v[i-1].val > dp[i][c])
            {
                dp[i][c] = dp[i-1][c - v[i-1].w].val + v[i-1].val;
                take[i][c] = true;
            }
        }
    }
    return {dp, take};
}
```

### Reconstruir Knapsack
```cpp
vi rebuild(vector<Item>items, vvb &take, int k)
{
    int n = items.size();
    vi select;
    for(int i = n; i >=1; i--)
    {
        if (take[i][k])
        {
            select.push_back(i-1);
            k -= items[i-1].w;
        }
    }
    reverse(all(select));
    return select;
}
```

**Ejemplo**

```cpp
vector<Item>items;
int capacity = 7;

auto [dp, take] = knapsack(items, take, capacity);
cout << "Max: "<< dp[dp.size()-1][capacity] << endl;


vi selected = rebuild(items, take, capacity);
for (auto idx : selected) cout << idx << endl;


```

## Elevator / Subset DP
- Mínimo número de viajes para subir personas con límite de peso.
- Subproblema: Para cada subconjunto `S`, calcular `(rides(S), last(S))`.

Código:
```cpp
struct Item
{
    int val, w;
};

pair<int,int> best[1<<N];

void solve_knapsack_mask(vector<Item>&v, int k)
{
    int n = v.size();
    best[0] = {1, 0}; // subconjunto vacío
    forn(s, 1, (1<<n))
    {
        best[s] = {n+1, 0};
        forn(p, 0, n)
        {
            if (s & (1<<p))
            {
                auto op = best[s ^ (1<<p)];
                if (op.second + v[p].w <= k)
                    op.second += v[p].w;
                else
                {
                    op.first++;
                    op.second = v[p].w;
                }
                best[s] = min(best[s], op);
            }
        }
    }
}
```

## Counting Tilings (Grid DP)
- Contar formas de llenar un grid `n x m` con piezas `1x2` y `2x1`.
- Subproblema: `count(k,x)` = formas de llenar filas `1..k` con estado `x` en fila `k`.

Idea:
- Representar cada fila como un estado.
- `count(k,x)` depende solo de `count(k-1,y)` donde `y` es compatible con `x`.
- Optimización: representar solo columnas con parte superior de los tiles verticales → 2^m estados.

Código:
```cpp
vi prev(1<<m,0), curr(1<<m,0);
prev[0] = 1;
forn(row, 1, n+1)
{
    for(int mask=0;mask<(1<<m);mask++)
    {
        for(int pmask=0;pmask<(1<<m);pmask++)
        {
            if(compatible(mask,pmask))
                curr[mask] += prev[pmask];
        }
    }
    swap(prev,curr);
    fill(all(curr), 0);
}
```
